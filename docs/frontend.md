## Принципы UX для метрик и рекомендаций

Цель UI — не “показать графики”, а привести пользователя к решению.

1) **Светофор** (зелёный/жёлтый/красный) должен быть консистентным с методологией.
2) **Explainability**: у каждого вывода — “почему так” + “что сделать дальше”.
3) **Quality flags**: если тест неполный/сомнительный — UI обязан показать предупреждение и ограничить уверенные рекомендации.
4) **Сценарии**: выводы должны резолвиться на 4 сценария (обучение/игры/программирование/офис).

## Поток данных

- FE → BE: запуск теста / загрузка артефактов / запрос результатов
- BE → FE: REST (итоговые результаты теста)
- BE → FE: SSE (live metrics)

Упрощённо:

    FE -> BE: start / upload / get results (REST)
    BE -> FE: live metrics (SSE)
    FE -> BE: history / profiles / configs (REST)

## “Результат теста” (минимальный контракт экрана)

Экран должен содержать:

- Сводка по 4 сценариям (score + краткий диагноз).
- “Узкое место” (CPU/GPU/RAM/Disk/thermal/stability) с объяснением.
- Блок “Качество данных” (quality flags).
- Рекомендации: “быстрое действие” (настройки/софт) и “апгрейд” (железо).
